使用正则表达式提取字符串中的数值

发表于 2018-11-30 | 分类于 java

工作中遇到需要提取字符串中的数值，借助网上的一个办法：

public class PatternTest1 {

       public static void main(String[] args){
           String s = "(1&(4|5))|10";

           //开始利用正则表达式
           String base = "[^0-9]";
           Pattern p = Pattern.compile(base);
        Matcher m = p.matcher(s);
        String nums = m.replaceAll("").trim();

           for(int i = 0; i < nums.length(); i++){
            Integer num = Integer.valueOf(String.valueOf(nums.charAt(i)));
               System.out.print(num + ", ");
        }    
    }
}

输出结果：1,4,5,1,0,

发现10分为(1,0)输出，所以字符串中有大于9的数值不能用这个方法。

下面的例子可以输出10：

public class PatternTest2 {

    public static void main(String[] args) {
        String s = "((1&(4|5))|10)";

        //下面开始利用正则表达式输出s中的数值
        String base = "\\d+";
        Pattern p = Pattern.compile(base);
        Matcher m = p.matcher(s);

        while(m.find()) {
            System.out.print(m.group() + ", ");
        }
    }
  }

输出结果：1, 4, 5, 10,

实际工作是需要把 s 转化成：（a1==true&&(a4==true||a5==true))||10==true

下面的代码是在 PatternTest2 的基础上进一步实现最终需求。

public class PatternTest3 {

    public static void main(String[] args) {
        String s = "((1&(4|5))|10)";
        s = s.replace("|", "||");
        s = s.replace("&", "&&");

        //下面开始利用正则表达式输出s中的数值
        String base = "\\d+";
        Pattern p = Pattern.compile(base);
        Matcher m = p.matcher(s);

        while(m.find()) {
            s.replace(m.group(), "a" + m.group() + "==true");
            System.out.print(m.group() + ", ");
        }
        System.out.println(s);
    }
}

java程序取出 mysql 数据库的信息

配置

代码

 public class TestJDBC {

       public static void main(String[] args) throws Exception{
        String driver = "com.mysql.jdbc.Driver";

        /*"jdbc:mysql": 通用的
         *
          *"localhost:3306/ceshi": localhost:主机名或ip地址; 3306:端口; ceshi:数据库的名字
            *
            *"?serverTimezone=UTC": (解决下面的错误，不要忘记加"?")
          * -->Caused by: com.mysql.cj.core.exceptions.InvalidConnectionAttributeException:
          *    The server time zone value '�й���׼ʱ��' is unrecognized or represents more than one time zone.

           *    to use a more specifc time zone value if you want to utilize time zone support.
         */

        String url = "jdbc:mysql://localhost:3306/ceshi?serverTimezone=UTC";
        String username="root";
            String password="zwy";

        //注册数据库驱动(反射：通过一个类名将这个数据反向加载到网页中)，driver值是固定的
        Class.forName(driver);

        //获得Connection数据库连接
        Connection conn = DriverManager.getConnection(url, username, password);

           //获得数据库连接执行平台
        Statement state = conn.createStatement();

        //sql语句, "ceshi_table": table名字
        String sql = "select * from ceshi_table";

        //通过执行平台执行查的操作并返回结果
        ResultSet set = state.executeQuery(sql);

        while(set.next()){
            System.out.println(set.getString("id"));
            System.out.println(set.getString("name"));
        }

        state.close();
        conn.close();
    }
}

trimAndSplit

发表于 2018-11-06 | 分类于 Java

工作中发现之前大佬写的一个通用方法，将 String 类转化为 Map 类。

代码如下：

public class StringToMap{
    public static void main(String[] args){
        Map<Integer, Integer> m = new HashMap<Integer, Integer>();

        String s = "1@2;3@4";
        String[] s1 = s.split(";");

        for(String s2 : s1){
            String[] s3 = s2.trim().split("@");
            int a = Integer.parseInt(s3[0]);
             int b = Integer.parseInt(s3[1]);
            m.put(a,b);
        }
    }
}

trim() 方法为去掉 String 字符串的首尾空格，自己在测试时将 trim() 方法去掉，即

String[] s3 = s2.split("@");

结果发现结果与去掉之前相同，为什么要加 trim() 方法呢，我又继续做测试，将字符串前后都加上空格，即

String s = " 1@2;3@4 ";

这时候在不加 trim() 方法，会发生 java.lang.NumberFormatException: For input string: “ 1” 错误，可知就是因为1前面有空格，使转换成数字类型时出错。

RemoveAllTest

发表于 2018-11-06 | 分类于 Java

今天看到一个例子，取得某一集合中另一个集合没有的元素，是先求得两个集合的交集，在取这个集合和交集的差集。我想直接取两个集合的差集是不是就可以，因此研究一下有关集合的 removeAll() 方法，A.removeAll(B) 方法是求两个集合的差集，这是在网上查找最多的说法，用我自己的话说为删除 A 集合中的元素，删除的元素是在 B 集合中也存在的。

public boolean removeAll(Collection<?> c){
    boolean modified = false;
    if(size() < c.size()){
        for(Iterator<?> i = c.iterator; i.hasNext(); ){
            modified |= remove(i.next());
        }
    }else{
        for(Iterator<?> i = iterator; i.hasnext(); ){
            if(c.contains(i.next){
                i.remove();
                modified = true;
            }
    }
}

由源码可知，当 A 集合的元素个数少于 B 集合的元素个数，遍历 B 集合，调用 remove() 方法，删除 A 集合中存在的元素；当 A 集合中的元素大于 B 集合中的元素，遍历 A 集合，判断 B 集合中是否存在这个元素，如果存在，就删除 A 集合中的这个元素。所以当一个集合调用这个方法与另一集合进行比较后，删除那些自己集合中存在，另一集合也存在的元素。

public class RemoveAllTest {

    public static void main(String[] args) {

        Set<Integer> set1 = new HashSet<Integer>();
        Set<Integer> set2 = new HashSet<Integer>();
        Set<Integer> set3 = new HashSet<Integer>();
        for(int i = 4; i < 10; i++) {
            set1.add(i);
            set3.add(i);
        }

        for(int i= 18; i >= 7; i--) {
            set2.add(i);
        }

        //经过测验可知，先取 set1 和 set2 的交集，再取这个交集和 set1 的差集的结果与直接取 set1 与 set2 的差集的结果相同(set1 与 set3 集合中元素相同）
        //set1.retainAll(set2);        
        //set3.removeAll(set1);

        set3.removeAll(set2);
        System.out.println(set3);
    }

}
