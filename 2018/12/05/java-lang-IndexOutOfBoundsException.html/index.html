java.lang.IndexOutOfBoundsException

发表于 2018-12-05 | 分类于 JAVA

java.lang.IndexOutOfBoundsException: index:2, size:1

当发生这个错误的时候很多时候是因为 get() 方法中的参数大于集合中元素的数量，但是 index 后面的 2 与 size 后面的 1 分别是什么意思呢？

经过试验得知：index 是 get() 方法输入的参数，size 是集合中元素的数量。

代码如下：

public class ListTest {

    public static void main(String[] args) {

        List<Integer> test1 = new ArrayList<Integer>();
        test1.add(1);
        System.out.println(test1.get(2));
    }

}

错误：
Exception in thread "main" java.lang.IndexOutOfBoundsException: Index: 2, Size: 1
at java.util.ArrayList.rangeCheck(Unknown Source)
at java.util.ArrayList.get(Unknown Source)
at test.ListTest.main(ListTest.java:12)


public class ListTest2 {

    public static void main(String[] args) {

        List<Integer> test2 = new ArrayList<Integer>();
        test2.add(1);
        test2.add(1);
        System.out.println(test2.get(2));
    }

}

错误：
Exception in thread "main" java.lang.IndexOutOfBoundsException: Index: 2, Size: 2
at java.util.ArrayList.rangeCheck(Unknown Source)
at java.util.ArrayList.get(Unknown Source)
at test.ListTest.main(ListTest.java:13)


public class ListTest3 {

    public static void main(String[] args) {

        List<Integer> test3 = new ArrayList<Integer>();
        test3.add(1);
        test3.set(2, 3); 
        System.out.println(test3.get(2));
    }

}

错误：
Exception in thread "main" java.lang.IndexOutOfBoundsException: Index: 2, Size: 1
at java.util.ArrayList.rangeCheck(Unknown Source)
at java.util.ArrayList.set(Unknown Source)
at test.ListTest3.main(ListTest3.java:12)

由 ListTest3 可得这个错误是在 test3.set(2, 3); 这行发生的，说明当你对 List 中某一个没有值的位置进行 set() 操作是错误的，也会发生 java.lang.IndexOutOfBoundsException。
